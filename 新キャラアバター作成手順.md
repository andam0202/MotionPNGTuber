# 新キャラクターでアバター（口パク＋まばたき＋Live2D風トラッキング）を作る手順

gura で確立した工程を、別キャラに適用するための実践手順書。
**口パクループ版**（声で口パク＋webカメラまばたき）と **Live2Dパペット版**
（see-through分解層を頭の向きで2.5D駆動）の2系統がある。両方とも素材は共通。

> 用語: `<CHAR>` = キャラのキー名（例: gura）。作業ディレクトリは
> `MotionPNGTuber/workspace/<CHAR>/`。WSLで実行（`env PULSE_SERVER=unix:/mnt/wslg/PulseServer uv run ...`）。

---

## 0. 前提環境（一度だけ）

| 要素 | 内容 |
|------|------|
| ComfyUI | Windows側で起動、WSLから `172.27.112.1:8001`（portproxy）。口/閉じ目のinpaint生成に使用 |
| see-through | `~/Documents/GitHub/see-through`（`.venv`構築済, GPU/torch2.8+cu128）。レイヤー分解に使用 |
| mediapipe-live | `~/Documents/GitHub/idoladeus/projects/mediapipe-live`。**Windows venv `.venv-win`**（mediapipe）と`models/face_landmarker.task`。webカメラ顔トラッキングに使用 |
| マイク | WSLから `--device 4`（RDPSource, PULSE_SERVER経由）。声の口パク・MFCC学習に使用 |
| webカメラ | WSLからは見えない。必ずWindows `.venv-win` 経由でUDP→WSL（WSL2とWindowsはlocalhost別スタック→WSL実IP宛に送信） |

## 1. 元素材を用意する

`workspace/<CHAR>/` に置く:
- `base_face.png` … 顔正面の立ち絵（1280x960推奨, 口が見える）。ComfyUI出力など。
- `loop.mp4` … 体のアイドルループ動画（30fps, 1280x960）。
- `loop_mouthless.mp4` … `loop.mp4` の口を消した版（`auto_erase_mouth.py`）。
- `mouth_track.npz` … 口の位置トラック（`auto_mouth_track_v2.py` または `face_track_anime_detector.py`）。

> 既存キャラの新規立ち上げは [project_motionpngtuber_setup] の手順（auto_mouth_track→auto_erase_mouth）を踏襲。

---

## 2. 口スプライト（あ/い/う/え/お…）

ComfyUIで立ち絵から口形状を生成（obsidian `comfyui_api/generate_mouth_sprites.py` →
`crop_mouth_sprites.py`）。結果は `workspace/<CHAR>/mouth/` に
`closed/half/aa/o/u/e/open.png`（194x194 RGBA）。
※ Live2Dパペットの音声口パクはこのスプライトを流用する。

---

## 3. 声のMFCC学習（口パクの母音判定）

```bash
cd MotionPNGTuber
env PULSE_SERVER=unix:/mnt/wslg/PulseServer uv run python train_vowel_mfcc.py --device 4 --reps 3
# あ/い/う/え/お を各3回録音 → vowel_mfcc.npz
```
- **有声フレームゲートが肝**（無音混入を除くと精度激増）。生録音は `vowel_mfcc_raw.npz` にキャッシュ、`--from-raw` で録り直さず再学習可。
- 確認: rep間クロス検証で90%+ なら良。

---

## 4.（任意）コマ数を増やす（60fps化）

```bash
ffmpeg -y -i workspace/<CHAR>/loop_mouthless.mp4 \
  -filter:v "minterpolate=fps=60:mi_mode=mci:mc_mode=aobmc:vsbmc=1" \
  -c:v libx264 -crf 17 -pix_fmt yuv420p workspace/<CHAR>/loop_mouthless_60.mp4
uv run python interp_track.py workspace/<CHAR>/mouth_track.npz \
  --out workspace/<CHAR>/mouth_track_60.npz --like-video workspace/<CHAR>/loop_mouthless_60.mp4
```
> WSLはGPU Vulkan不可のためRIFE不可。ffmpeg minterpolateを使う。トラックは動画と同フレーム数に補間必須（`get_quad`が剰余参照のため）。

## 5. see-through で目を分離（のっぺらぼう＋目レイヤー）

```bash
cp workspace/<CHAR>/base_face.png ~/Documents/GitHub/see-through/data/input/<CHAR>_base.png
cd ~/Documents/GitHub/see-through
.venv/bin/python inference/scripts/inference_psd.py \
  --srcp data/input/<CHAR>_base.png --save_dir data/output --save_to_psd
# 出力は実際は data/output_marigold/<CHAR>_base/ に layers(1280x1280)
```
- **重い（実測~49分、拡散ベース）**。GPU稼働を確認しつつ待つ。HFモデル未キャッシュなら数GB DL。
- 得られる主な層: `face.png`(目/鼻/口を消したのっぺらぼう肌), `eyewhite/irides/eyelash`(開き目), `mouth`等。
- **座標変換**: 層は1280x1280（入力1280x960を **y+160 パディング**, scale1）。base整合は `[160:1120]` で切る。

---

## 6. 目アセット生成（eyeless / open / closed）

`build_eye_assets.py` 冒頭の **`ST_DIR` を `<CHAR>_base` に差し替え**てから:

```bash
uv run python build_eye_assets.py --out-dir workspace/<CHAR>/eye
# eyeless.png(目消し肌パッチ) / open.png(開き目) / closed.png(閉じ目) を生成
```
- **閉じ目はComfyUIだとまつ毛が薄い**。`build_eye_assets.py` は **see-throughの`eyelash`層を下げて(sh35)閉じまつ毛**を構成（ComfyUI不要・左右対称・画風一致）。
- 目の左右box（縦ワイプ範囲・左右分割）はマスク重心から算出（`eye_mask_st`の左右中心）。

---

## 7.（パペット用）のっぺらぼうループを焼き込み

```bash
# eyeless.png を loop_mouthless_60.mp4 全フレームに合成 → loop_eyeless_60.mp4
# (build_eye_assets後、簡易スクリプトで cv2 合成。gura はこの手順で作成)
```
ベースが目無しになるので「のっぺらぼうの上に目を載せる」構成が曖昧さなく成立。

## 8A. 口パクループ版を動かす

```bash
# Windows: webカメラ→まばたき送信
bash start_eye_tracker.sh        # pose_server(.venv-win)を WSL実IP:5006 へ。要 --host自動

# WSL: ランタイム（口=声MFCC, 目=webカメラ, 60fps）
env PULSE_SERVER=unix:/mnt/wslg/PulseServer uv run python -u loop_lipsync_runtime_patched_emotion_auto.py \
  --loop-video workspace/<CHAR>/loop_eyeless_60.mp4 \
  --mouth-dir workspace/<CHAR>/mouth --track workspace/<CHAR>/mouth_track_60.npz \
  --device 4 --lipsync-mode mfcc \
  --eye-track udp --eye-sprite-dir workspace/<CHAR>/eye --eye-base-eyeless
```

## 8B. Live2Dパペット版を動かす（頭の向きで2.5D駆動）

```bash
# WSL: パペット（口=声MFCC, 頭/目=webカメラ）
env PULSE_SERVER=unix:/mnt/wslg/PulseServer uv run python -u live2d_puppet.py --port 5007
# Windows: 顔ポーズ送信（頭yaw/pitch/roll + blink/gaze + 口blendshape）
<.venv-win>/python.exe face_pose_server.py --host=<WSL_IP> --port 5007 --cam 0 --preview
```
- 既定 `--mouth-source audio`（マイクMFCC母音→口形割当）。`camera`でjawOpen連続開閉に切替可。
- まばたき: 閉じ目イラストへ離散切替（`--blink-thresh`+ヒステリシス）。

## 9. キャラ別に差し替える値（gura固定値→要変更）

現状はgura固定値が各所にある。新キャラでは以下を更新（将来 project.toml 化推奨）:

| ファイル | 変数 | 内容 |
|----------|------|------|
| `build_eye_assets.py` | `ST_DIR` | see-through出力dir（`<CHAR>_base`） |
| `gen_eye_sprites.py` | `BASE_POSITIVE`/`BASE_NEGATIVE` | キャラのプロンプト（gawr gura等→新キャラ） |
| `gen_eye_sprites.py` | `EYES`, `CKPT` | 目box（実位置）, checkpoint |
| `motionpngtuber/eye_blink.py` | `DEFAULT_LEFT/RIGHT` | 目box（縦ワイプ・左右分割） |
| `live2d_puppet.py` | `ST`, `MOUTH_DIR`, `MOUTH_ANCHOR`, `EYE_CY`, closed.pngパス, `Z_ORDER`深度 | 各パス・口位置・目中心・層深度 |
| `live2d_puppet.py` | `pick_mouth` 閾値 | お/う判定（話者により mouthFunnel が弱い→pucker大小で判別。実測HUDで調整） |
| `face_pose_server.py` | `MODEL` | face_landmarker.task のパス |

> 目boxの実位置は `eye_mask_st`（see-through目層α和）の左右重心から算出する（gura: 左cx468 / 右cx838, cy~436）。

## ハマりどころ（重要）

- **WSLにカメラ無し**（`/dev/video*`無）。webカメラは必ず Windows `.venv-win` 経由でUDP→WSL実IP。
- **色がネガ反転**して見えたら、`imread`(BGR)層を`cvtColor(RGB2BGR)`して二重変換していないか確認（表示はBGRのまま）。
- **see-through層は1280x1280**（y+160）。base(1280x960)整合は`[160:1120]`。
- **閉じ目/口のComfyUI inpaint**は小領域＋肌上だと薄く/左右非対称になりがち。閉じ目はeyelash層流用が確実。口は既存の良質スプライト流用。
- `set -e` なshellでは pkill/grep の非ゼロでスクリプトが中断する。停止は `ps|awk|while kill`、起動はpkillと別コマンドに。
- see-through分解は重い（~49分）＋他プロセスとのHF帯域競合に注意。
